Student answers question

Use Case Title: Student answers question
Use Case ID: UC2
Primary Actor: Student
Goal in Context: A student wants to answer a question posted on the system.
Scope: Educational – The overall answering mechanism to questions posted by contributing students as seen by all of the Exam Overflow community.

Stakeholders and Interests:
  • Student: Student wants to answer a question.
Preconditions:
  • The students account has not been barred from the system.
Main Success Scenario:
  1. System: Displays all questions which have been asked by students
  2. Student: Selects the question they wish to answer.
  3. System: Displays the answers already posted (if any) for the selected question. A text field is also displayed for new answers to be posted.
  4. Student: Types their answer into the text field provided, clicks the ‘Post Answer’ button.
  5. System: Validates the answer provided by the student.
  6. System: Saves answer to system, displays a message confirming that the answer has been saved.

Extensions:

1a. System No question exists
2a. Student can only post a question.
4b. Student enters an answer that already exists.
4c. Student provides an answer containing illegal characters.
6b. System does not save answer, system displays an error message.
6c. System does not save answer, system displays an error message.

Post conditions:
  • The answer has been saved and can be viewed by other students.
  • Students can now like the new answer.

Priority: High
Releases: One and may require updates.
Frequency of Use: Many times each day
Channel to Primary Actor: System
