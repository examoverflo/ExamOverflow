Seek Answer to question

Use Case Title: Student seeks answer to question
Use Case ID: UC1
Primary Actor: Student
Goal in Context: A student requires an answer to a question. The student posts this question with a view to receiving suggested answers from the community of fellow students.
Scope: Educational – The overall question posting forum as seen by all of the Exam Overflow community.

Stakeholders and Interests:
  • Student: Student wants to receive an answer to a question.
Preconditions:
  • The students account has not been barred from the system.
Main Success Scenario:
  1. Student: Submits a question to the system
  2. System: Checks the database of questions in order to ensure that duplicate questions are not being posted on the system.
  3. System: The system displays a confirm message to the user that the exam question has been successfully uploaded.

Extensions:

1a. Student submits a question that already exists on the system.
2a. System recognises that the question submitted already exists.
3a. System does not upload the students’ question. Rather the system directs the student to the duplicate question originally posted on the forum and the associated answers.

Post conditions:
  • The question has been stored in the system.
  • The question cannot be asked again in the future.
  • Students can now provide answers for the Exam Question.

Priority: Medium
Releases: One
Frequency of Use: Many times after completion of college exams
Channel to Primary Actor: System
