Use Case Title: Post an Exam Question

Use Case ID: 3

Primary Actor : Student

Preconditions :The student is on the Post Exam Question Page

Main Success Scenario :

  1. System: The system requests the year for the Exam Question.
  2. Student: The student enters the year for the Exam Question.
  3. System: The system requests the module for the Exam Question.
  4. Student: The student enters the module for the Exam Question.
  5. System: The system requests the topic for the Exam Question.
  6. Student: The student enters the topic for the Exam Question.
  7. System: The system requests the Exam Question text from the student
  8. Student: The student enters the Exam Question Text.
  9. Student: The student clicks the "Submit" button
  10. System: The system saves the exam question entered.
  11. System: The system displays a confirm message to the user that the exam question has been successfully uploaded.
Extensions :
  • 2a The student fails to enter the year of the Exam Question
  • 4a The student fails to enter the module for the Exam Question
  • 6a The student fails to enter the topic for the Exam Question
  • 10a The Exam Question for the year, module and topic entered has already been asked by another student.

    • Postcondition : The Exam Question is stored. The Exam Question cannot be asked again in the future.

    Priority: High

    Releases: None

    Frequency of Use : Three times a week

    Channel to Primary Actor: System
