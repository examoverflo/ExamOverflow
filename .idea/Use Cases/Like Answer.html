Like answer

Primary actor: Student

Goal in context: The student wishes to express their approval of an answer posted to the ExamOverflow by a fellow student. This answer has most likely been most useful for the student posting the like.
Scope: Educational â€“ The overall likes mechanism which orders answers by their popularity as provided by the community of contributing students.
Level: Summary
Stakeholders and Interests
Community: Group of students who contribute to ExamOverflow
Students: Wish to contribute answers for their fellow students.
They also like the answers provided. This is to give each answer a weighting, the higher this weighting (number of likes) the more accurate and/or helpful the answer is perceived to be.
Preconditions: The answer must be posted before it can be liked. Each answer can only be liked by the same user once.
Minimal Guarantees: An answer receives some likes, this verifies that the content of the answer is accurate and helpful.
Success Guarantees: An answer that receives a high number of likes will be highly rated and thus should be a trusted source of information for the rest of the community.
Trigger: Student likes an answer
Main Success Scenario:
1. Student: Student looks through the answers provided by other students
2. Student: Student decides to like an answer and so clicks the like button.
3. System: The system checks if the student has already liked the answer
4. System: The system updates the likes for the associated answer. Extensions: 3a. The user attempts to like an answer that they have already liked. Only one like per answer will be recorded and thus an appropriate error message will be displayed to the screen.
Technology and Data Variations List: None.
Priority: High.
Releases: One.
Response Time: Instantaneous.
Frequency of use: Various.
