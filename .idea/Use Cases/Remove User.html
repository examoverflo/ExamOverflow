Remove User - Fully Dressed Use Case

Primary Actor: Administrator

Secondary Actor: Student

Scope: An administrator is a student who ranks among the top 10 users with regard to likes.

Goal in Context: An administrator wishes to remove a disruptive student from the system

Stakeholders and Interests

  • Administrator: Wants to remove a student and all contrinution from the system, in a simple and easy to use manner.
  • Student: Wants to receive a message detailing why they have been removed from the system
  • Other Students: Wants to receive email if their answer has been removed, because another users question has been removed.

Precondition: Administrator has administration rights.

Minimal Guarantess: All answers posted by the Student have been removed from the system.

Trigger: Administrator decides a Student needs to be removed from the system

Main Success Scenario

  1. Administrator: Administrator click button to remove a student
  2. System: System verifies that the Administrator has administration rights
  3. System: System removes all answers given by the Student
  4. System: System removes all answers to question posted by the Student
  5. System: System sends email to all Other Students who have answered questions posted by the Student
  6. System: System removes all questions asked by the Student
  7. System: System removes the Students account from the system
  8. System: System creates a template email for the Administrator
  9. Administrator: Administrator completes the email, specifying the reasons for the Students removal
  10. System: System sends the completed email to the Student

Alternative Flow

  • 2a: Administrator doesn't have administration rights
  • 2b: System displays meessage saying Administrator does't have administration rights
  • 3a: An error occurs when trying to delete the answers
  • 3b: System displays an error message saying the answers could not be removed. System does not continue.
  • 4a: An error occurs when trying to delete answers for question posted by the Student
  • 4b: System displays an error message and the system ends the process
  • 4c: The system rollsback previous changes
  • 5a: An error occurs when sending the email to Other Students
  • 5b: System displays error message and ends the process
  • 5c: The system rollsback previous changes
  • 6a: An error occurs when trying to remove questions asked by the Student
  • 6b: System displays an error message and the process is ended
  • 6c: The system rollsback previous changes
  • 7a: An error occurs when trying to remove the Students account
  • 7b: System displays an error message and the process is ended
  • 7c: The system rollsback previous changes
  • 8a: An error occurs when the template email is generated
  • 8b: System displays an errror message and the process is ended
  • 8c: The system rollsback previous changes
  • 10a:An error occurs when the system tries to send the eamil to the Students
  • 10b:The system displays an error message and the process is ended
  • 10c: The system rollsback previous changes

Post Condition: All answers and questions posted by the Student have been deleted as well as the users account. The Student has received a email detailing the reasons for their removal.Other Students have received emails if their answers have been deleted in the process

Priority: Low-Medium

Releases: Several

Frequency of Use: Low, maybe once a month

Channels to Primary Actor: Application

Channels to Secondary Actor: Application, Email
